## Ход работы
1) была поднята виртуальная машина на ядре убунту сервера через виртуалбокс, так как у меня винда, и произведено ссш подключение к машине из терминала для удобства
2) [скрипт на го](./main.go) с использованием библиотеки syscall для работы с системными вызовами, охватывающий все что по заданию надо было реализовать
```text
логика разделена на два этапа:

Фаза Parent (функция run)
Отвечает за подготовку ресурсов хост-системы перед изоляцией:

Конфигурация: Считывает id, hostname и command из config.json.

Файловая система (OverlayFS): Создает структуру в /var/lib/mini-docker/{id} и монтирует слои:

lowerdir: Базовый образ (ReadOnly).

upperdir: Слой записи для изменений.

workdir: Служебная директория для атомарных операций.

merged: Итоговая точка монтирования.

Изоляция: Перезапускает процесс через /proc/self/exe с флагами CLONE_NEWUTS, CLONE_NEWPID и CLONE_NEWNS.

Фаза Child (функция child)
Выполняется уже внутри новых пространств имен:

Смена корня: Выполняет syscall.Chroot в директорию merged и переходит в новый корень через os.Chdir("/").

Идентификация: Устанавливает hostname из конфигурации.

Интерфейс ядра: Монтирует /proc для корректной работы системных утилит (позволяет видеть только локальные процессы).

Исполнение: Запускает целевую команду (например, /bin/sh).
```
