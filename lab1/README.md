# Mini-Docker: Утилита контейнеризации 

Проект реализует минимальный рантайм контейнеризации на языке Go. Утилита создает изолированную среду для процессов, используя возможности ядра Linux: **Namespaces** (изоляция) и **OverlayFS** (многослойная файловая система).

## Быстрый старт

> **Требования:** ОС Linux (ядро 5.x+). Все команды выполняются из директории лабораторной работы.

### 1. Клонирование и переход в рабочую область
```bash
git clone https://github.com/aiaanad/devops_facultative_labs.git
cd devops_facultative_labs/lab1
```

### 2. Подготовка зависимостей и RootFS
Для работы контейнера нужен базовый образ ОС (в нашем случае Alpine Linux). Мы скачиваем его и распаковываем в папку `lowdir`, которая станет слоем **ReadOnly**.

```bash
# Обновление или установка системных утилит (если нет)
sudo apt update && sudo apt install -y golang-go wget tar

# Подготовка базового слоя (lowerdir)
mkdir -p lowdir && wget -qO- https://dl-cdn.alpinelinux.org/alpine/v3.18/releases/x86_64/alpine-minirootfs-3.18.4-x86_64.tar.gz | tar -xz -C ./lowdir
```

### 3. Конфигурация (config.json)
Убедитесь, что в текущей папке (`lab1/`) присутствует файл конфигурации. 

### 4. Запуск контейнера
Для управления пространствами имен и монтирования требуются права суперпользователя.

```bash
# Сборка и запуск
sudo go run main.go run
```
